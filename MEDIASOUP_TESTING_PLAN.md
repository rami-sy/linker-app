# 🧪 Mediasoup Call System - خطة الاختبار الشاملة

## 🎯 الهدف
التحقق من جميع التحسينات المنجزة والتأكد من عمل النظام بشكل صحيح.

---

## 📋 قائمة التحقق الأساسية

### ✅ **1. Logger System**
- [ ] **تسجيل موحد**: جميع الرسائل تستخدم `logger` بدلاً من `console.log`
- [ ] **مستويات مختلفة**: `debug`, `info`, `warn`, `error` تعمل بشكل صحيح
- [ ] **دوال خاصة**: `callEvent`, `deviceEvent`, `roomEvent`, `streamEvent` تعمل
- [ ] **تحكم في التفعيل**: يمكن إلغاء/تفعيل التسجيل عبر متغير البيئة
- [ ] **0 console.log متبقية**: لا توجد `console.log` في ملفات Mediasoup

### ✅ **2. Finite State Machine (FSM)**
- [ ] **حالات صحيحة**: `idle`, `inviting`, `ringing`, `joining`, `producing`, `consuming`, `inCall`, `leaving`, `ended`, `error`
- [ ] **انتقالات صحيحة**: جميع الدوال تستخدم `stateMachine.transition()`
- [ ] **حماية من العمليات غير المسموحة**: لا يمكن بدء مكالمة أثناء مكالمة جارية
- [ ] **تتبع دقيق**: `callState` متاح في الواجهة ويعكس الحالة الحقيقية
- [ ] **انتقالات مرنة**: يدعم جميع السيناريوهات (group calls, leaving, ending, etc.)

### ✅ **3. Guard Manager**
- [ ] **دوال الحماية**: `canJoin()`, `canLeave()`, `canStartCall()` تعمل بشكل صحيح
- [ ] **إدارة مركزية**: `setJoining()`, `setLeaving()`, `setHasLeftRoom()` تعمل
- [ ] **إعادة تعيين ذكية**: `resetForNewRoom()`, `resetAll()` تعمل
- [ ] **منع التكرار**: لا يمكن استدعاء `joinRoom` أو `leaveRoom` عدة مرات
- [ ] **توافق مع الكود القديم**: مزامنة تلقائية مع legacy refs

### ✅ **4. Error Codes System**
- [ ] **رموز موحدة**: `DEVICE_NOT_FOUND`, `PERMISSION_DENIED`, `STREAM_CREATION_FAILED`, إلخ
- [ ] **رسائل واضحة**: كل رمز خطأ له رسالة واضحة للمستخدم
- [ ] **تصنيف موحد**: `createError()` و `normalizeError()` تعمل بشكل صحيح
- [ ] **معالجة متسقة**: جميع الأخطاء تُعالج بنفس الطريقة

### ✅ **5. Stream Manager**
- [ ] **إنشاء Stream**: `createMediaStream()` يعمل مع جميع التكوينات
- [ ] **إيقاف Stream**: `stopMediaStream()` يوقف جميع الـ tracks بشكل صحيح
- [ ] **بناء Constraints**: `buildMediaConstraints()` يبني constraints صحيحة
- [ ] **التحقق من الأجهزة**: `validateDevices()` يتحقق من توفر الأجهزة
- [ ] **معالجة الأخطاء**: جميع الأخطاء تُعالج بـ Error Codes

### ✅ **6. Device Manager**
- [ ] **كشف الأجهزة**: `detectDevices()` يكشف الأجهزة المتاحة
- [ ] **طلب الأذونات**: `requestDevicePermissions()` يطلب الأذونات
- [ ] **إعادة المحاولة**: `requestDevicePermissionsWithRetry()` يعيد المحاولة
- [ ] **تخزين مؤقت**: Device cache يعمل بشكل صحيح
- [ ] **اختيار الأجهزة**: `getSelectedDevices()` يختار الأجهزة المناسبة

### ✅ **7. Race Condition Handling**
- [ ] **منع التداخلات**: `cancelledCallsRef` يمنع التداخلات
- [ ] **إدارة الرفض**: `rejectedBySetRef` يدير الرفض في المجموعات
- [ ] **عدّاد المستقبلين**: `recipientsCountRef` يحسب المستقبلين بدقة
- [ ] **عدّاد المشاركين**: `peersJoinedCountRef` يحسب المشاركين الفعليين
- [ ] **منع الإلغاء المبكر**: لا يتم إرسال `callCancelled` إلا في الحالة الصحيحة

### ✅ **8. Server Side Improvements**
- [ ] **ملخص الدعوة**: `callInviteSummary` يرسل عدد المستقبلين
- [ ] **إنهاء المكالمة**: `endCall` ينهي المكالمة للجميع
- [ ] **إشعار انتهاء المكالمة**: `callEnded` يُرسل للمشاركين
- [ ] **إدارة المجموعات**: منطق المجموعات يعمل بشكل صحيح

### ✅ **9. Socket Listeners Integration**
- [ ] **استمعات موحدة**: جميع listeners تستخدم `logger`
- [ ] **FSM integration**: جميع listeners تستخدم FSM transitions
- [ ] **Guard protection**: جميع listeners تستخدم Guard Manager
- [ ] **تنظيف منظم**: جميع listeners يتم تنظيفها بشكل صحيح

### ✅ **10. API Surface Cleanup**
- [ ] **واجهة نظيفة**: `useMediasoup` يعرض واجهة منظمة
- [ ] **دوال واضحة**: `startCall`, `acceptCall`, `rejectCall`, `leaveRoom`, `endCallForAll`
- [ ] **حالات منظمة**: State منظم في مجموعات منطقية
- [ ] **إجراءات واضحة**: Actions منظمة ومفهومة

### ✅ **11. Reset All Function**
- [ ] **تنظيف شامل**: `resetAll()` ينظف جميع الموارد
- [ ] **إيقاف Streams**: يوقف جميع MediaStreams
- [ ] **إيقاف Producers/Consumers**: يوقف جميع Producers والـ Consumers
- [ ] **إيقاف Transports**: يوقف جميع Transports
- [ ] **إعادة تعيين الحالات**: يعيد تعيين جميع الحالات
- [ ] **إعادة تعيين FSM**: يعيد تعيين FSM إلى الحالة الأولية
- [ ] **معالجة الأخطاء**: يتعامل مع الأخطاء بشكل صحيح
