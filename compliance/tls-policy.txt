TLS Encryption Policy — Linker App (linker.land)
Date: May 2026

How TLS 1.2 or greater is enabled for data in transit:

1. The Linker application is served exclusively over HTTPS using TLS 1.2 and TLS 1.3.
   - SSL/TLS certificates are issued and managed by Let's Encrypt via Certbot.
   - Nginx is configured to enforce HTTPS on all endpoints.
   - HTTP traffic (port 80) is automatically redirected to HTTPS (port 443).

2. Platform Data is NEVER transmitted without encryption in transit.
   - All API calls between the client and server use HTTPS only.
   - SSL 2.0 and SSL 3.0 are never used.
   - TLS 1.0 and TLS 1.1 are disabled on the server.

3. Nginx TLS configuration enforces:
   ssl_protocols TLSv1.2 TLSv1.3;
   ssl_prefer_server_ciphers on;
